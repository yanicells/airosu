# airosu V1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Browser game that plays standard osu! beatmaps with webcam hand-tracking aim (Manual and Relax modes), deployed free on Vercel (the user connects the repo in the Vercel dashboard manually — do not add deploy automation or use any Vercel CLI).

**Architecture:** Fully static client-side app. Five modules with hard boundaries: `cv/` (camera → smoothed cursor), `beatmap/` (.osz → internal model), `game/` (pure-TS clock/judging/scoring — no DOM imports), `render/` (PixiJS stage), `ui/` (React shell). `game/` is TDD'd; render/ui verified manually via dev server.

**Tech Stack:** Vite, TypeScript, React 18+, PixiJS v8, `osu-parsers` + `osu-classes`, `fflate`, `@mediapipe/tasks-vision` (HandLandmarker), Web Audio API, Vitest.

**Spec:** `docs/superpowers/specs/2026-07-04-airosu-design.md` — read it first.

## Global Constraints

- All processing client-side; camera frames never leave the device.
- `src/game/` and `src/cv/filters.ts`, `src/cv/calibration.ts`, `src/cv/palm.ts` must be pure TS: no imports from `pixi.js`, `react`, `@mediapipe/*`, or DOM APIs (types like `AudioContext` behind interfaces).
- osu! playfield coordinate space is 512×384; internal model and game logic use it everywhere. Rendering maps it to screen.
- The `.osz` fixture (`444335 HO-KAGO TEA TIME - Kira Kira Days.osz`) is for local tests only — it must never be copied into `public/` or the deployed bundle.
- Verify current versions of all npm packages at install time (`npm view <pkg> version`); pin exact versions. If a documented API differs from this plan, trust the installed library and note the deviation in the PR description.
- All commit messages: conventional commits (`feat:`, `fix:`, `test:`, `chore:`, `docs:`).
- Node 20+.

### Task 4.3: Game session — circles, manual + relax (pure)

**Files:**
- Create: `src/game/session.ts`
- Test: `src/game/session.test.ts`

**Interfaces:**
- Consumes: model (2.1), judge (4.1), score (4.2).
- Produces:

```ts
export interface HitEvent { objectIndex: number; judgment: Judgment; at: Vec2 }
export interface SessionState {
  score: ScoreState; activeObjects: number[]; // indices visible (time - preempt .. judged)
  finished: boolean;
}
export class GameSession {
  constructor(map: LoadedBeatmap, settings: Settings);
  /** advance clock; relax mode auto-judges due objects; returns events since last tick */
  tick(timeMs: number, cursor: Vec2 | null): HitEvent[];
  /** manual mode key press */
  press(timeMs: number, cursor: Vec2 | null): HitEvent | null;
  readonly state: SessionState;
  /** AR preempt ms: AR<5 → 1200+600*(5-AR)/5 ; AR≥5 → 1200-750*(AR-5)/5 */
  preemptMs(): number;
}
```

Rules: objects judged in order (earliest unjudged is the target). **Relax:** on `tick`, when `timeMs >= object.time` and cursor in radius → judgment from `judgeTiming(0 …)` = 300 (relax is timing-perfect; quality only reduced by distance: outer 30% of forgiving radius → 100); cursor null or out of radius when `timeMs > object.time + w50` → miss. **Manual:** `press` judges earliest active object if cursor in radius and timing not 'ignore'; else null (no penalty for stray presses); unpressed object past `time + w50` → miss on next tick. Circles only in this task (sliders/spinners judged as circles at their head; refined in M5).

Manual: `press(1005, onObject)` → 300; `press(1005, farAway)` → null; no press, tick(1200) → miss. `finished` true after last object judged.
